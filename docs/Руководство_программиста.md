# Руководство программиста SmartMedical

Проект SmartMedical разворачивается как приложение Next.js с PostgreSQL, Prisma и NextAuth. Для запуска требуется Node.js 20+, npm 10+, PostgreSQL 14+ и доступ к Cloudinary для работы с изображениями.

После клонирования репозитория и установки зависимостей командой `npm install` необходимо создать файл `.env` в корне проекта. Обязательные переменные: `DATABASE_URL`, `NEXTAUTH_URL`, `NEXTAUTH_SECRET`, `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`. Дополнительно, при необходимости, задаются `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `OPENROUTER_API_KEY`, `NEXT_PUBLIC_SITE_URL`.

Инициализация базы выполняется через `npm run setup` либо последовательно командами `npm run prisma:migrate`, `npm run prisma:generate`, `npm run prisma:seed`, `npm run prisma:normalize:new-structure`. Для просмотра данных используется `npm run prisma:studio`.

Локальный запуск выполняется командой `npm run dev`, после чего приложение доступно по адресу `http://localhost:3000`. В штатной разработке применяются команды `npm run lint`, `npm run build`, `npm start`, а для сопровождения БД используются `npm run prisma:migrate:dev`, `npm run prisma:migrate`, `npm run prisma:generate` и, при необходимости полного сброса локальных данных, `npm run prisma:reset`.

Ролевой доступ проверяется сервером по данным пользователя в таблице `patients`: `USER` для пользовательских разделов, `OPERATOR` для операторских чатов, `CHIEF_DOCTOR` для обработки писем и связанных административных операций, `ADMIN` для полного доступа к админ-функциям.

Для production-развертывания требуется подготовить корректный `.env` с боевыми значениями, проверить `NEXTAUTH_URL`, выполнить `npm run build`, применить миграции `npm run prisma:migrate`, запустить сервер командой `npm start` и провести контрольную проверку авторизации, API и основных административных сценариев.
